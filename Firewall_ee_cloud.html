Firewalling in EE Cloud

This diagram shows the firewalling related to the EE Cloud

In EE cloud your cloud instances are grouped into "security groups". There is a virtual firewall between your security group and the outside world which you are able to configure freely. The settings of this security group are your responsibility and are the only thing between your instances and the other instances in the cloud. For this reason you must ensure that they are set correctly and allow exactly the traffic you want to allow.

NSN firewall

Between EE Cloud environments in DHN and the rest of the NSN networks there are IT firewalls. The aim is that most traffic that you would normally need with the intranet will be allowed by default by these firewalls. If you find that some basic traffic is missing then please make a firewall request together with the EE Cloud team to add that traffic for all security groups.

For information about production environments and which network segment they are located, please see the page Which cloud suits my purpose .

Special rules

If you need special rules for your security group only, you should allocate yourself an "elastic IP" which is an IP address that will remain with your account independent of the addresses of your instances. You may then make firewall requests to NSN IT Firewall team for that IP address.

When you have finished with the elastic IP address, please request the NSN firewall team to remove your special rules and then afterwards deallocate it.
