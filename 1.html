Controller on Cloud PoC

Case

This is a PoC work on migrating the radio controller to cloud. In this PoC, we will try some alternatives on the following aspects:

  1. The cloud platform
  2. The possiblity to reuse currently VSP/VGP components
  3. The fast packet handling performance on cloud based platform.
  4. The impact on the current site solution
Reason for change

In Liquid core, Many PoC projects have been done, some of them are demostrated with our customer, they are very successful and got possitive feedback. The next slide tell us the major eventsabout cloud in core network in NSN. For controller, we have not started. But this is the time to start We can get many experience from liquid core and many assets to reuse. If we want to migrate controller to cloud, TT is a very critical area, Fast packet handling is a very big challenge on telecloud, this part is mainly covered by TT area. And a lot of transport adaption work are all changed compare cloud with the dedicate hardware currently. So we need to join this PoC work and make important contribution to help our controller on cloud to be successful.

chang scale

NSN have 23000 BSCs and 3000RNCs running all over the world. And check the market trend in the 2014, 2015 and future, 2G and 3G will increase not so quickly but continuously. Customer have CAPEX and OPEX pressure if they need to buy more and more hardware device and manage them in current way. Cloud is an opportunity, Putting the controller to cloud will save the CAPEX and OPEX obviously.

Return of Investment
  • Cost of investment
    Currently, we have planed 4 months work with a 6 people team. total investiment might be 6*3*22*8*20=63360
  • Benefit
    We will have oportunity to demostrate the controller on cloud with our customer earlier than our customer. Just as the liquid core has did with IFA and Vodafone. The return may can't be measured in 6-12months, but if the product comes out from the PoC, The benefit of our PoC work is obviously and valueable then.
